# 06. Governança de documentação e Git

## Estratégia
- branch principal: `main`;
- branch de desenvolvimento: `develop`;
- branches curtas por funcionalidade;
- pull requests com revisão;
- versionamento das entregas.

## Documentos obrigatórios
- visão da solução;
- arquitetura;
- fluxos;
- plano de testes;
- validação;
- backlog.

## Boas práticas
- commits pequenos e rastreáveis;
- issues por melhoria;
- changelog por release;
- critérios de aceite explícitos.
