# 07. Backlog do MVP

## Sprint 1
- estruturar arquitetura;
- mapear intenções principais;
- definir fluxos;
- publicar protótipo inicial.

## Sprint 2
- rodar teste com usuários;
- ajustar UX e textos;
- incluir métricas básicas;
- validar com time técnico.

## Sprint 3
- validar com negócio;
- preparar rollout controlado;
- documentar aprendizados;
- priorizar roadmap seguinte.
